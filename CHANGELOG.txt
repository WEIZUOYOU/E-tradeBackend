一、需求背景
-----------
原有商品状态已定义 "0-待审核"，但缺少完整的审核流程：
- 无审核列表接口（管理员无法查看待审核商品）
- 无审核通过/驳回接口
- 无驳回原因字段
- 无审核人、审核时间记录

二、新增状态值
-----------
商品 status 新增: 4 = 审核不通过
完整状态: 0-待审核, 1-在售, 2-已下架, 3-已售出, 4-审核不通过

六、举报处理模块 + 消息通知模块 - 变更日志 (2026-05-13)
============================================================

一、举报处理新增接口
-----------
GET  /api/v1/trade/report/list?status=&page=&size=  - 举报列表(分页+状态筛选)
PUT  /api/v1/trade/report/{id}/handle               - 处理举报(下架商品+封号+通知)
PUT  /api/v1/trade/report/{id}/dismiss               - 驳回举报(通知举报人)

处理举报时自动执行:
  1. 下架被举报商品(status→2)
  2. 冻结卖家账号(status→0)
  3. 向卖家发送通知(商品被举报下架)
  4. 向举报人发送通知(举报已处理)

二、消息通知模块接口
-----------
GET  /api/notification/list?page=&size=    - 当前用户通知列表
GET  /api/notification/unread-count        - 未读通知数量
PUT  /api/notification/{id}/read           - 标记单条已读
PUT  /api/notification/read-all            - 全部标记已读

通知类型:
  0-系统通知   1-举报处理结果   2-审核结果

微信模板消息:
  WeChatNotifyUtil.java 已创建骨架，配置微信公众号AppId/AppSecret后即可启用真实推送

七、数据库迁移(如已有库)
-----------
-- 已有字段补充:
ALTER TABLE product
  ADD COLUMN review_reason VARCHAR(500) COMMENT '审核驳回原因',
  ADD COLUMN reviewer_id INT COMMENT '审核人ID',
  ADD COLUMN reviewed_time DATETIME COMMENT '审核时间';

-- 举报表新增字段:
ALTER TABLE report
  ADD COLUMN handler_id BIGINT COMMENT '处理人ID',
  ADD COLUMN handle_result VARCHAR(255) COMMENT '处理结果说明',
  ADD COLUMN handle_time DATETIME COMMENT '处理时间';

-- 通知表:
CREATE TABLE IF NOT EXISTS `notification` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `user_id` BIGINT NOT NULL,
    `type` TINYINT DEFAULT 0 COMMENT '0-系统通知, 1-举报处理, 2-审核结果',
    `title` VARCHAR(100) NOT NULL,
    `content` TEXT,
    `related_id` BIGINT,
    `is_read` TINYINT DEFAULT 0,
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_read (user_id, is_read),
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
